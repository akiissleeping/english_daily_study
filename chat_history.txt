作ったアプリの音声が聞こえないんだけど、何がたりないの


----------------------------------------------------------------
[CLAUDE]
----------------------------------------------------------------
■ 原因
  音声再生機能が実装されていなかった。
  スクリプトのテキストのみで、読み上げる仕組みがなかった。

■ 解決策の選択肢
  方法                      メリット              デメリット
  Web Speech API（内蔵TTS） 無料・APIキー不要      声質はロボット系
  Google TTS / Amazon Polly 自然な音声            APIキー・課金が必要
  音声ファイル（mp3）        最高品質              準備が大変

  → ブラウザ内蔵の Web Speech API で対応

■ 追加した機能

  useSpeech カスタムhook
    - 文ごとに分割して順番に再生
    - 再生中の文をインデックス管理

  SpeakBtn コンポーネント
    - 単語・フレーズ用の小さな 🔊 ボタン
    - クリックで即時読み上げ

  ScriptPlayer コンポーネント
    - ▶ / ⏹ の再生・停止ボタン
    - 速度調整：遅い(0.7x) / 普通(0.85x) / 速い(1.0x) / 最速(1.2x)
    - 再生中は現在の文を黄色ハイライト
    - 各文クリックで個別再生
    - 「💡 各文をクリックして個別再生できます」注釈

■ 各セクションへの適用
  語彙     : 各単語の横に 🔊 ボタン・例文の横にも 🔊 ボタン
  リスニング: スクリプト全文プレイヤー（穴埋め前に内容をつかめる）
  トレーニング:
    Step1 リピーティング  → プレイヤー + スクリプト表示
    Step2 音読           → プレイヤー + スクリプト表示
    Step3 ディクテーション→ プレイヤー + テキストエリア（スクリプト非表示）
    Step4 オーバーラッピング→ プレイヤー + スクリプト表示
    Step5 シャドーイング  → プレイヤー + スクリプトぼかし（クリックで表示）

  コミット: "Add Web Speech API TTS audio to all sections"

■ 原因
  「▶ スタート」はタイマー（5分カウントダウン）のボタン。
  音声再生は ScriptPlayer の ▶ ボタンが別にある。
  両方が同じ紫色・▶ アイコンで混同しやすいUIだった。

■ 修正内容

  タイマーボタン
    変更前: 紫色の「▶ スタート」
    変更後: グレーの「⏱ タイマー開始」「⏸ タイマー停止」

  音声プレイヤー
    変更前: 目立たないデザイン
    変更後:
      - 青い枠（border: 2px solid var(--primary-light)）
      - 「🔊 音声再生プレイヤー」ヘッダーラベル追加
      - ▶ ボタンを大きく（42px）・影付きで強調
